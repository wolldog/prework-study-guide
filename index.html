HTML

  • Hypertext Markup Language or HTML is one of 3 key languages used in web development.
  • HTML provides the framework for the webpage.
  • HTML is made up of elements, each with a distinct role. Some elements contain content, others provide information.
  • The 'head' element provides information or 'metadata' for the browser to ensure the page renders correctly.
  • Information in the 'head' element is not visible to the user.
  • '!DOCTYPE html', the first line in an HTML document, is a declaration to tell the browser that is is looking at an HTML document.
  • The 'title' element is required and will be displayed in the tab at the top of the page.
  • The 'meta' element defines metadata such as the character set, description, keywords, author and viewport.
  • The 'link' element tells the browser where to find any external resouce required. For example an external stylesheet.
  • The 'body' element sets out the structure and contains the visible content of the webpage.
  • The h1 through h6 elements represent levels of heading. 'h1' is the largest and most prominent. There should only be one 'h1' in a document.
  • 'ul' and 'ol' elements represent unordered (bulleted) and ordered (numbered) lists. The 'li' element is a list item.
  • An element refers to the entire element; tags and content. Tags are just what appears inside the angle bracket.
  • Semantic HTML elements are elements whose name describes the content.
  • It is best practice to use semantic elements wherever possible as it makes your HTML easier to read.
  • Elements that are nested inside other elements are know as 'child' elements.
  • The element that holds all the other elements is called the 'parent'.
  • A well structured HTML document will be user friendly and easy to navigate.
  • An HTML attribute defines additional information about an element.
  • HTML elements are case sensitive.

CSS

  • CSS styling can be achieved in 3 ways; inline, internal or external.
  • A CSS rule has a 'selector' and a 'declaration'.
  • The selector identifies which element the style should apply to.
  • The declaration specifes a property and assigns a value to the property.
  • Each HTML element is essentially a box with a margin (outside) and padding (inside).
  • A '*' selector can be used to style every element on a page.
  • If two or more elements share the same styling you can combine their rules.
  • If using a seperate document to add style you need to incorporate a link in the 'head' element of the HTML doc.
  • When you don't want to style all instances of an element in the same way, use a 'class' attribute.
  • Choose a word that will best describe where the style should be used, then precede it with a full stop, ie. '.card'
  • Incorporate the class attribute in the opening tag of the element.

Git

  • Git is the most commonly used version control software among developers.
  • Git allows multiple developers to work on a project without overwriting each others work.
  • Git tracks all changes and makes it possible to revert to an earlier version of code if an error is introduced.
  • You can use Git on your local machine to track changes but it is usually used in conjunction with a remote cloud based repository like GitHub.
  • To access Git we used the command line.
  • Git works using a system called Git branching.
  • The files and folders that contain the code and assets for a project reside on the 'main' branch.
  • When a developer wants to contribute to a project, they copy the main branch to a feature branch. They make changes on the feature branch, never the main branch.
  • Once changes have been vetted and reviewed in the feature branch, the files are merged back into the main branch.
  • git status checks what branch we are currently on and list any current changes.
  • git checkout [branch name] moves you from the branch you are on to another named branch.
  • git checkout -b [branch-name] creates a new branch and switches to it.
  • git add [filename] adds the named file to a staging area ready for the next commit.
  • git commit captures a snapshot of the currently staged changes.
  • git pull origin main pulls a fresh copy of the original files from the origin main branch.
  • git commit -m "Description of changes" commits (saves) the changes you have added to the staging area along with a description.
  • git push origin [feature branch name]pushes changes in our locally stored feature branch to a remote branch.

JavaScript

  • JavaScript (JS) is a high-level dynamic programming language that allows users to interact with websites.
  • Application Programming Interfaces or APIs built atop the core JavaScript language provide enormous functionality in a plug n play fashion.
  • Variables are containers that store values.
  • Variables can hold values from many data types; String, Number, Boolean, Array and Object.
  • JavaScript is case sensitive so MyVariable is not the same as myvariable.
  • Comments can be inserted in code to assist understanding. /* Anything between the slashes including over line breaks is ignored */
  • An 'operator' is a mathematical symbol that produces a result based on two variables.
  • Conditionals are code structures used to test if an expression is true or false for example; if.....else.
  • Functions are a way of packaging functionality that you wish to reuse.
  • Some functions are built into the browser but you can write your own.
  • Events are code structures that listen for activity in the browser, for example, a 'click' event executes a function when you click on something.
